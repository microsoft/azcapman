# T-2.1 — Reorder initialization in all three quota scripts

## Goal

Swap the initialization order so subscriptions and locations are resolved BEFORE default SKU discovery. This is a prerequisite for the Catalogs API migration since the API requires a subscription ID and location.

## Why

Currently all three scripts resolve SKUs first (line ~392), then subscriptions (line ~397), then locations (line ~402). The Azure Catalogs API requires an authenticated subscription context and a location parameter. Without reordering, the new `Get-SKUDetails` can't call the API.

## Files

- `scripts/quota/Get-AzVMQuotaUsage.ps1`
- `scripts/quota/Get-AzVMQuotaUsageCli.ps1`
- `scripts/quota/Show-AzVMQuotaReport.ps1`

## Acceptance criteria

1. In all 3 scripts, `$SubscriptionIds` is populated BEFORE `Get-SKUDetails` is called
2. In all 3 scripts, `$Locations` is populated BEFORE `Get-SKUDetails` is called
3. The existing behavior when `-SKUs` is explicitly provided remains unchanged (no API call)
4. The existing behavior when `-SubscriptionIds` or `-Locations` are explicitly provided remains unchanged
5. All 3 scripts parse without PowerShell syntax errors after the change
6. No other logic is modified — this is a pure reordering task

## Reference

Current order (Get-AzVMQuotaUsage.ps1 lines 392-406):
```powershell
if ($SKUs.Count -eq 0) { $SKUs = Get-SKUDetails }
if ($SubscriptionIds.Count -eq 0) { $SubscriptionIds = Get-SubscriptionIds }
if ($Locations.Count -eq 0) { $Locations = Get-Locations | Sort-Object }
```

Required order:
```powershell
if ($SubscriptionIds.Count -eq 0) { $SubscriptionIds = Get-SubscriptionIds }
if ($Locations.Count -eq 0) { $Locations = Get-Locations | Sort-Object }
if ($SKUs.Count -eq 0) { $SKUs = Get-SKUDetails }
```
