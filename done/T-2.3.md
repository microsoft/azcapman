# T-2.3 — Implement catalog data fetcher for CLI script

## Goal

Create a `Get-VMCatalogData` function in the CLI script that calls the Azure Reservations Catalog REST API via `az rest` with pagination support.

## Why

The CLI script (`Get-AzVMQuotaUsageCli.ps1`) uses `az` CLI exclusively. `Get-AzReservationCatalog` is an Az PowerShell cmdlet, and `az reservations catalog show` is a preview extension. Using `az rest` is consistent with the script's existing pattern (it already uses `az rest` for zone peering at line ~219) and avoids preview extension dependencies.

## Files

- `scripts/quota/Get-AzVMQuotaUsageCli.ps1`

## API details

Same REST endpoint as T-2.2, but called via:
```powershell
$response = az rest --method GET --url $uri -o json | ConvertFrom-Json
```

Pagination via `nextLink` property in response.

## Acceptance criteria

1. Function `Get-VMCatalogData` is added to the CLI script
2. Uses `az rest` — no `az reservations` extension dependency
3. Accepts `$SubscriptionId` and `$Location` parameters
4. Implements pagination via `nextLink` (do/while loop)
5. Returns array of `[PSCustomObject]@{ InstanceSizeFlexibilityGroup; ArmSkuName; Ratio }`
6. Extracts `ArmSkuName` from `.name`, group from `ReservationsAutofitGroup` skuProperty, ratio from `ReservationsAutofitRatio` skuProperty
7. Handles items missing ISF properties gracefully (skip, don't error)
8. Function includes comment-based help
9. Script parses without syntax errors after the change
