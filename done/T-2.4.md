# T-2.4 — Wire up Get-SKUDetails with API-first and CSV fallback

## Goal

Modify `Get-SKUDetails` in all 3 scripts to use `Get-VMCatalogData` as the primary data source, with the existing CSV download as an explicit fallback on API failure.

## Why

This completes the Catalogs API migration while preserving backward compatibility. The CSV fallback handles edge cases: `Microsoft.Capacity` RP not registered, missing RBAC permissions, or API unavailability.

## Files

- `scripts/quota/Get-AzVMQuotaUsage.ps1`
- `scripts/quota/Get-AzVMQuotaUsageCli.ps1`
- `scripts/quota/Show-AzVMQuotaReport.ps1`

## New parameter

Add to all 3 scripts' `param()` block:
```powershell
[Parameter(Mandatory = $false, HelpMessage = "Azure region to query for VM catalog data")]
[string]$CatalogLocation = "eastus"
```

## Implementation pattern

```powershell
function Get-SKUDetails {
    try {
        Write-Host "Querying Azure Catalogs API for VM SKU details"
        $catalogData = Get-VMCatalogData -SubscriptionId $SubscriptionIds[0] -Location $CatalogLocation
        $skuNames = ($catalogData | Select-Object -Property ArmSkuName -Unique |
            Where-Object { $_.ArmSkuName -notlike "*sql*" }).ArmSkuName
        if ($skuNames.Count -eq 0) { throw "Catalogs API returned zero VM SKUs" }
        Write-Host "Retrieved $($skuNames.Count) unique VM SKUs from Catalogs API"
        return $skuNames
    }
    catch {
        Write-Warning "Catalogs API failed: $($_.Exception.Message)"
        Write-Warning "Falling back to CSV download from $MeterDataUri"
        # existing CSV download logic
        [string]$meterDataFile = $MeterDataUri.Split('/')[-1]
        Invoke-WebRequest -Uri $MeterDataUri -OutFile $meterDataFile
        $meterData = Get-Content $meterDataFile | ConvertFrom-Csv
        return ($meterData | Select-Object -Property ArmSkuName -Unique |
            Where-Object { $_.ArmSkuName -notlike "*sql*" }).ArmSkuName
    }
}
```

## Acceptance criteria

1. All 3 scripts have `$CatalogLocation` parameter with default `eastus`
2. `Get-SKUDetails` tries Catalogs API first using `$SubscriptionIds[0]` and `$CatalogLocation`
3. On API success, returns unique non-SQL ArmSkuName values from catalog data
4. On API failure, emits `Write-Warning` (never silent) and falls back to CSV download
5. CSV fallback uses existing `$MeterDataUri` parameter and download logic
6. Return type remains `string[]` — no contract change
7. Depends on T-2.1 (init order) and T-2.2/T-2.3 (Get-VMCatalogData) being complete
8. All 3 scripts parse without syntax errors
9. `$MeterDataUri` help text updated to indicate it's a fallback source
