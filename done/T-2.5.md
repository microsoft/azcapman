# T-2.5 — Update documentation for Catalogs API migration

## Goal

Update all documentation artifacts to reflect the Catalogs API as the primary data source for VM SKU discovery, with CSV as a deprecated fallback.

## Why

Docs must match the new behavior. Users need to understand the auth requirements, the `$CatalogLocation` parameter, and that `$MeterDataUri` is now a fallback.

## Files

- `scripts/quota/Get-AzVMQuotaUsage.ps1` (comment-based help)
- `scripts/quota/Get-AzVMQuotaUsageCli.ps1` (comment-based help)
- `scripts/quota/Show-AzVMQuotaReport.ps1` (comment-based help)
- `docs/operations/tools-scripts/show-azvmquotareport.md`
- `.gitignore` (keep both `isfratioblob.csv` and `AutofitComboMeterData.csv` entries)

## Acceptance criteria

1. All 3 scripts' `.PARAMETER MeterDataUri` help text indicates this is a deprecated fallback
2. All 3 scripts have `.PARAMETER CatalogLocation` help text
3. All 3 scripts' `.DESCRIPTION` mentions the Catalogs API as the primary SKU data source
4. All 3 scripts' `.NOTES` mentions `Microsoft.Capacity/catalogs/read` permission requirement
5. `docs/operations/tools-scripts/show-azvmquotareport.md` troubleshooting section updated for API errors
6. No prohibited words per style guide (utilize, leverage, powerful, seamless, etc.)
7. Sentence-style capitalization used throughout
